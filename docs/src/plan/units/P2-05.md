# P2-05: Create walrus-telegram Crate

## Goal

Create the walrus-telegram crate skeleton with TelegramChannel struct
implementing the Channel trait interface.

## Depends On

Phase 1 (P1-03 — Channel trait, ChannelMessage, Platform).

## Design Decisions

- Uses reqwest directly for Telegram Bot API calls (DD#2). No teloxide.
- TelegramChannel fields: bot_token (`CompactString`), client (`reqwest::Client`),
  poll_timeout (u64, seconds, default 30).
- Implements Channel trait with stub methods (connect returns empty stream,
  send returns Ok). Actual implementation in P2-06.
- Telegram Bot API base URL: `https://api.telegram.org/bot{token}/`.

## Files

- **CREATE** `crates/telegram/Cargo.toml` — depends on walrus-core, reqwest,
  serde, serde_json, futures-core, async-stream, compact_str, anyhow, tokio
- **CREATE** `crates/telegram/src/lib.rs` — TelegramChannel struct, Channel stubs
- **MODIFY** `Cargo.toml` (workspace root) — add `crates/telegram` member

## Tests

- `telegram_channel_platform` — verify platform() returns Platform::Telegram
- `telegram_channel_construction` — verify struct builds with token and client

## Done When

- [x] `cargo check --workspace` passes
- [x] TelegramChannel compiles and implements Channel trait
