# P1-02: Add MemoryEntry and RecallOptions Types

## Goal

Add supporting types referenced by Memory trait's async methods.

## Depends On

P1-01. P1-01 introduces stub MemoryEntry and RecallOptions if needed for trait
signatures to compile. This unit completes them with all fields, derives, and
re-exports.

## Design Decisions

- Both types live in `crates/core/src/memory.rs`.
- Timestamps are `u64` (unix seconds) — no chrono dependency in walrus-core.
- Metadata is `Option<serde_json::Value>`. Add `serde_json.workspace = true` to walrus-core deps.
- RecallOptions has `Default` (limit 0, no range, no threshold).

## Files

- **MODIFY** `crates/core/src/memory.rs` — add/refine MemoryEntry and RecallOptions
- **MODIFY** `crates/core/src/lib.rs` — re-export both types

## Key Fields

- **MemoryEntry**: key, value, metadata (`Option<Value>`), created_at, accessed_at, access_count, embedding (`Option<Vec<f32>>`). Derives: Debug, Clone.
- **RecallOptions**: limit (usize), time_range (`Option<(u64, u64)>`), relevance_threshold (`Option<f32>`). Derives: Debug, Clone, Default.

## Tests

- `memory_entry_default` — verify default field values
- `recall_options_default` — verify Default derives (limit 0, no range, no threshold)
- `memory_entry_clone` — verify Clone works

## Done When

- [ ] `cargo check --workspace` passes
- [ ] Both types re-exported from walrus-core
