# P6-06: Hub CLI Wiring (`walrus hub` Subcommand)

## Goal

Wire all hub operations into the CLI as `walrus hub <subcommand>`,
dispatching to `walrus-hub` library functions.

## Depends On

P6-03 (search/info), P6-04 (install/remove), P6-05 (update/sources).

## Design Decisions

- `Hub` variant added to CLI Command enum (P4-03). Contains a `HubCommand`
  sub-enum with variants: Search, Info, Install, Remove, Update, Upgrade,
  AddSource, RemoveSource.
- `walrus hub search <query>` — prints formatted search results.
- `walrus hub info <name>` — prints manifest details with install status.
- `walrus hub install <name> [--force]` — discovers workspace via
  `hub::find_workspace_root()`, installs resource.
- `walrus hub remove <name>` — discovers workspace, removes resource.
- `walrus hub update` — syncs all sources, reports available updates.
- `walrus hub upgrade [name]` — upgrades one or all resources.
- `walrus hub add-source <name> <owner/repo>` — adds hub source.
- `walrus hub remove-source <name>` — removes hub source.
- Install/remove/upgrade require a workspace (error if no `walrus.toml`).
  Search/info/update work without a workspace.
