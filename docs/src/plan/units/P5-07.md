# P5-07: Marketing Agent Example Workspace

## Goal

Create a complete example workspace demonstrating the marketing agent
use case. Validates the workspace format and serves as reference.

## Depends On

P5-02 (`walrus init` exists to validate workspace format).

## Design Decisions

- Lives in `examples/marketing/` as a workspace directory.
- `walrus.toml`: gateway settings, sqlite memory at `data/memory.db`,
  auth with a placeholder API key.
- `agents/marketing.toml`: marketing agent with tools `["browser_*",
  "remember"]`, skill_tags `["marketing", "twitter"]`, system prompt
  for community engagement.
- `skills/`: four skill files (twitter_auth.md, twitter_read.md,
  twitter_post.md, marketing_strategy.md) with TOML frontmatter.
- `mcp/playwright.toml`: Playwright MCP server via npx, headless mode.
- `cron/daily_tweet.toml`: daily at 10am, message template for the
  full read-learn-post cycle.
- `.gitignore`: ignores `data/`.
- `README.md`: setup, prerequisites, first-run walkthrough, monitoring.
