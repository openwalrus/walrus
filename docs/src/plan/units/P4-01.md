# P4-01: walrus-client Crate + WalrusClient Struct

## Goal

Create the walrus-client crate skeleton with WalrusClient struct and
ClientConfig. This is the shared client library used by CLI and other
platforms (mobile, desktop) to connect to a walrus-gateway.

## Depends On

Phase 3 (P3-02 — walrus-protocol crate must exist).

## Design Decisions

- WalrusClient is a config holder. Connection logic comes in P4-02.
- ClientConfig: gateway_url (`CompactString`), auth_token (`Option<CompactString>`).
- Depends on walrus-protocol for wire types, not on walrus-gateway.
- No async code in this unit — just types and builder.

## Files

- **MODIFY** `app/client/Cargo.toml` — depends on walrus-protocol, tokio,
  tokio-tungstenite, futures-core, anyhow, bytes, compact_str
- **MODIFY** `app/client/src/lib.rs` — WalrusClient, ClientConfig

## Tests

- `client_config_defaults` — verify default config construction
- `client_builder` — verify builder-style API
